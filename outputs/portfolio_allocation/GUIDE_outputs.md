# GUIDE_outputs.md

## Purpose

This folder stores artifacts generated by the portfolio-allocation stage.

Inputs come from:

- `outputs/correlation_analysis/selected_merged.csv` unless `--tickers` is supplied
- `data/raw/volume_screen.csv` for liquidity-derived market weights
- ClickHouse daily close history for the optimization sample

## Main Files

- `allocation_weights.csv`
- `allocation_summary.csv`
- `allocation_recommended.csv`
- `allocation_weights_chart.png`
- `allocation_cumulative.png`
- `allocation_efficient_frontier.png`
- `method_lab_summary.csv`
- source-suffixed method-lab snapshots such as `allocation_summary_merged.csv`

## Summary Fields

`allocation_summary.csv` includes:

- `ann_return`
- `total_return`
- `cagr`
- `ann_vol`
- `sharpe`
- `max_drawdown`
- `calmar_ratio`
- `effective_n`

The `calmar_ratio` field uses `cagr / max_drawdown`.
