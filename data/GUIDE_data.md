# GUIDE_data.md

## Part 1: Conceptual Explanation

This folder separates base inputs from generated artifacts.

- `raw/` holds the shared ETF universe CSV and the shared daily parquet.
- `external/`, `interim/`, `processed/`, and `cache/` are reserved for future
  data staging so the root layout stays stable as the project grows.

## Part 2: Code Reference

- `data/raw/volume_screen.csv`
  - Generated by `src/data_pipeline/screen.py`

- `data/raw/daily_close_volume_screened_2016_2025.parquet`
  - Generated by `src/data_pipeline/export_daily_data.py`
